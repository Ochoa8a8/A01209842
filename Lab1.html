Algunas tecnologías

URL El URL o Uniform Resource Locator, permite localizar recursos en Internet con el fin de facilitar la navegación. Es decir, es una referencia a un recurso del Internet. Un URL contiene dos principales componentes, el identificador de protocolo e indentificador de recurso. Un ejemplo de URL sería "http://ejemplo.com" en donde "http" sería el identificador de protocolo y "ejemplo.com" el del recurso [1].
MIME El MIME o Multi-Purpose Internet Mail Extensions es una extensión del protocolo original de correos de Internet. Éste permite intercambiar distintos tipos de archivos de Internet, como lo son: audio, videos, imagenes, entre otros [2].

Métodos y códigos de errores

HTTP Estos errores se envían al navegador si ocurre un problema al acceder a una página web. Cada error está representado por 3 digitos en donde el primero representa una clase del código de estado. Ej. Para un error con código 4XX, en donde el primer dígito es 4, indica que hay un error del lado del cliente [3].
XML El XML o Extensible Markup Language es un formato universal para datos y documentos estructurados. Los archivos de este tipo tienen una extensión xml. El formato XML utiliza etiquetas (similar al HTML) para estructurar los datos del documento [4]. A partir del XML es posible comunicarse con otras aplicaciones de diferentes plataformas sin que importe el origen de la información en común [5].
XHTML El XHTML o Extensible Hyper Text Markup Language sirve como un etiquetado más estricto que HTML lo que permite mejorar la interpretación de la información, sin importar el dispositivo que se utiliza [6].
HTML5 Es la versión más reciente de HTML con nuevos elementos, atributos y comportamientos. Fue diseñado para ser utilizado por los desarrolladores de la Open Web y contiene mejoras en cuanto a las versiones anteriores, como lo son: la semántica, conectividad, rendimiento, gráficos y efectos 2D/3D, entre otras cosas [7]
JS JavaScript es un lenguaje script multi-paradigma. Es comúnmente conocido como el lenguaje de script para páginas web aunque también es usado en muchos entornos sin navegador [8].
php El PHP o Hypertext Preprocessor es un lenguaje script que permite generar contenido dinámico en una página, manejar los archivos en el servidor enviar y recibir cookies, administrar datos en su base de datos, cifrar datos, entre otras cosas. Puede ser ejecutado en varias plataformas (Windows, Linux, Unix, Mac OS X, etc.) [9].
jsp JSP o Java Server Pages es una tecnología que permite a los desarolladores y diseñadores web desarrollar rápidamente y mantener fácilmente páginas web dinámicas y con gran contenido de información referentes a sistemas de empresas existentes [10].
Ajax AJAX o Asynchronous JavaScript and XML es una nueva técnica que permite crear aplicaciones web mejores, más rápidas y más interactivas utilizando XML, HTML, CSS y Java Script [11]

Preguntas a responder:

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

    2. Interet es el medio por el que se accede a la web, es decir, Internet es como una gran red que permite la comunicación y la web es un sistema en donde se almacena la información a la que accedemos.

  2. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    3. Estos métodos se utilizan para indicar una acción que se desea realizar para un recurso determinado [12].

    • GET: Se utiliza para pedir un recurso
    • HEAD: Funciona igual que GET solo que no obtiene el cuerpo de la respuesta
    • POST: Envíar o guardar un recurso en un servidor
    • PUT: Actualiza el recurso de destino
    • PATCH: Modifica parcialmente un recurso
    • DELETE: Borra el recurso
  3. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    4. Se utilizaría el método POST ya que este permite envíar al servidor.

  4. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    5. El método utilizado sería el GET ya que éste permite solicitar un recurso en específico.

Ciclo de vida y desarrollo de los sistemas de información

Los sistemas de información pasan por una serie de etapas a lo largo de su vida. Estas etapas son las siguientes:

  • Planificación: En esta primera fase se debe estudiar muy bien el proyecto, para así poder determinar el ámbito del mismo, un estudio de viabilidad, el análisis de los riesgos una estimación del costo del proyecto, entre otras cosas [18].
  • Análisis: En esta etapa se debe de definir qué es lo que se quiere lograr con el sistema. Resulta importante definir los requerimientos funcionales y no funcionales Es muy importante esta etapa ya que si no se sabe con exactitud cuales son las características que debe tener el sistema, resultará imposible hacer un buen diseño posteriormente [18].
  • Diseño: Para este punto se decide cómo es que se van a implementar los distintos requerimientos. Se estudian las diversas alternativas para conocer cuales serán las más eficientes para dicho sistema. En esta fase se hace un modelado representando las características de como construir el sistema [20].
  • Implementación: Es crucial que para esta etapa ya se tenga una excelente comprensión de lo que hará el sistema y el cómo lograrlo (las etapas anteriores). En esta fase se deben seleccionar las herramientas adecuadas así como el lenguaje con el que se construirá el sistema. Una vez hecho esto se podrá empezar a programar procurando siempre seguir las buenas prácticas [21].
  • Pruebas: La etapa de pruebas es muy importante ya que en la mayoría de las ocasiones se tendrán errores inclusive si se siguen muy bien los pasos anteriores. Esta fase ayuda a encontrar los posibles errores que hayan pasado desapercibidos para que así puedan ser reparados. Todo esto se deberá hacer antes de mostrar el producto al usuario final. Existen distintas pruebas, mismas que pueden ser clasificadas como pruebas de unidad, de integración, alfa, beta, etre otras [22].
  • Instalación o despliegue: En este momento es cuando se pone el sistema en funcionamiento. Para esto hay que planificar el entorno en el que el sistema se debe ejecutar tanto hardware como software [23].
  • Uso y mantenimiento: El mantento del sistema resultará en ser la etapa más larga de la vida del sitema, pues será aproximadamente 60% del ciclo. Éste incluye 3 principales tareas: eliminar los defectos que se detecten, adaptarlo a las nuevas necesidades y añadir nuevas funcionalidades [24].
